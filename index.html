IceFireDB
IceFireDB is a DB storage and retrieval protocol built for web3.0,Help applications quickly access distributed data networks.

Decentralized database storage
and retrieval Protocol

Multiple storage engine support

Support LSM disk, OSS, IPFS、CRDT、IPFS-LOG underlying storage.

Multi-communication protocol support

Support RESP, SQL communication protocols, support KV, Strings, Hashes, Lists, Sorted Sets, Sets, SQL, PubSub data structures.

Multiple Consensus Network Support

A single data center supports high availability of RAFT data, and supports CRDT and IPFS-LOG data consensus mechanisms across regions,IPC and tendermint consensus mechanisms are being supported.

Support for multiple data application scenarios

It supports web2 database storage scenarios, supports data decentralization scenarios from web2 to web3, and supports web3 database storage and retrieval requirements.

Projects
IceFireDB

IceFireDB is a database built for web3 and web2, and strives to fill the gap between web2 and web3 with the help of a friendly database experience, making web3 application data storage more convenient, and making web2 applications easier to access the blockchain.

IceFireDB-SQLite

IceFireDB-SQLite database is a decentralized SQLite database. Provide a convenient mechanism to build a global distributed database system. Support users to write data to IceFireDB using MySQL protocol.

IceFireDB-SQLProxy

IceFireDB-SQLProxy helps decentralize traditional SQL database data. Provide a convenient mechanism to build a globally distributed storage system with automatic networking.

IceFireDB-CRDT-KV

Decentralized kv storage engine,support decentralized P2P networking, data synchronization and consistency between nodes.

IceFireDB-IPFS-LOG

A distributed immutable, operation-based conflict-free replication data structure that relies on ipfs to store data and merges each peer node data based on pubsub conflict-free.
